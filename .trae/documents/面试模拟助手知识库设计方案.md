## 🎯 核心问题：是否需要多个 Pinecone Index？


### ✅ 方案三：单 Index + 多 Namespace + 元数据过滤（强烈推荐）⭐

```
Pinecone Index: interview-assistant-index
├── Namespace: official-java-backend (官方 Java 后端题库)
│   ├── 元数据：{category: "java-basic", difficulty: "easy"}
│   ├── 元数据：{category: "java-collection", difficulty: "medium"}
│   ├── 元数据：{category: "mysql", difficulty: "hard"}
│   └── ...
├── Namespace: official-frontend (官方前端题库)
│   ├── 元数据：{category: "html", difficulty: "easy"}
│   ├── 元数据：{category: "vue", difficulty: "medium"}
│   └── ...
├── Namespace: user-resume-user1 (用户 1 的简历)
├── Namespace: user-resume-user2 (用户 2 的简历)
└── ...
```

**优点**:
- ✅ **成本最低**: 只需 1 个 Index = $30/月
- ✅ **管理简单**: 统一配置，易于维护
- ✅ **检索高效**: 支持按 Namespace + 元数据组合过滤
- ✅ **灵活扩展**: 新增题库只需添加新的 Namespace 或元数据
- ✅ **用户隔离**: 每个用户的简历独立 Namespace

#### 2.1 批量导入题库到 Pinecone

@Component
public class QuestionDataLoader {

    @Autowired
    private EmbeddingModel embeddingModel;

    @Autowired
    private EmbeddingStore<TextSegment> embeddingStore;

    /**
     * 初始化官方题库
     */
    @PostConstruct
    public void initOfficialQuestions() {
        // Java 后端题库
        loadQuestionsFromPdf(
            "src/main/resources/content/java 后端/Java 基础面试题速记通关版.pdf",
            "official-java-backend",
            "java-basic",
            "easy"
        );

        loadQuestionsFromPdf(
            "src/main/resources/content/java 后端/MySQL 面试题速记通关版.pdf",
            "official-java-backend",
            "mysql",
            "medium"
        );

        // 前端题库
        loadQuestionsFromPdf(
            "src/main/resources/content/前端/Vue 基础面试题速记通关版.pdf",
            "official-frontend",
            "vue",
            "medium"
        );

        // ... 加载其他题库
    }

    /**
     * 从 PDF 文件加载面试题
     */
    private void loadQuestionsFromPdf(
            String pdfPath,
            String namespace,
            String category,
            String difficulty) {

        try {
            // 1. 加载 PDF 文档
            Document document = FileSystemDocumentLoader.loadDocument(
                pdfPath,
                new ApachePdfBoxDocumentParser()
            );

            // 2. 分割文档（按题目分割）
            DocumentSplitter splitter = new DocumentByParagraphSplitter(300, 30);
            List<TextSegment> segments = splitter.split(document);

            // 3. 为每个片段添加元数据
            EmbeddingStore<TextSegment> store = getStoreByNamespace(namespace);

            for (TextSegment segment : segments) {
                // 添加元数据
                segment.metadata().put("category", category);
                segment.metadata().put("difficulty", difficulty);
                segment.metadata().put("source", new File(pdfPath).getName());
                segment.metadata().put("type", "official");

                // 4. 向量化并存储
                Response<Embedding> response = embeddingModel.embed(segment);
                store.add(response.content(), segment);
            }

            log.info("成功加载 {} 道题到 {}: {}", segments.size(), namespace, category);

        } catch (Exception e) {
            log.error("加载题库失败：{}", pdfPath, e);
        }
    }

    private EmbeddingStore<TextSegment> getStoreByNamespace(String namespace) {
        return PineconeEmbeddingStore.builder()
                .apiKey(System.getenv("PINECONE_API_KEY"))
                .index("interview-assistant-index")
                .nameSpace(namespace)
                .build();
    }
}

### 步骤 4: 智能检索器实现

#### 4.1 创建面试专用的 ContentRetriever

```java
@Component
public class InterviewContentRetriever implements ContentRetriever {

    @Autowired
    private EmbeddingModel embeddingModel;

    /**
     * 检索面试相关知识点
     * 
     * @param query 查询文本
     * @param position 面试岗位
     * @param difficulty 难度级别
     * @param userId 用户 ID（用于检索用户简历）
     */
    public List<Content> retrieveForInterview(
            String query,
            String position,
            String difficulty,
            String userId) {

        // 1. 将查询向量化
        Embedding queryEmbedding = embeddingModel.embed(query).content();

        // 2. 确定要检索的 Namespace
        String officialNamespace = getNamespaceByPosition(position);

        // 3. 从官方题库检索
        EmbeddingStore<TextSegment> officialStore = getStoreByNamespace(officialNamespace);
        
        EmbeddingSearchRequest officialRequest = EmbeddingSearchRequest.builder()
                .queryEmbedding(queryEmbedding)
                .maxResults(5)
                .minScore(0.7)
                .filter(buildMetadataFilter(position, difficulty))  // 按难度过滤
                .build();

        List<EmbeddingMatch<TextSegment>> officialMatches = 
            officialStore.search(officialRequest).matches();

        // 4. 从用户简历检索（如果有）
        List<EmbeddingMatch<TextSegment>> resumeMatches = new ArrayList<>();
        if (userId != null) {
            EmbeddingStore<TextSegment> resumeStore = getStoreByNamespace("user-resume-" + userId);
            
            EmbeddingSearchRequest resumeRequest = EmbeddingSearchRequest.builder()
                    .queryEmbedding(queryEmbedding)
                    .maxResults(3)
                    .minScore(0.6)
                    .build();

            resumeMatches = resumeStore.search(resumeRequest).matches();
        }

        // 5. 合并结果
        List<Content> contents = new ArrayList<>();
        
        // 添加官方题库内容
        for (EmbeddingMatch<TextSegment> match : officialMatches) {
            contents.add(Content.from(
                "[知识点] " + match.embedded().text()
            ));
        }

        // 添加用户简历内容
        for (EmbeddingMatch<TextSegment> match : resumeMatches) {
            contents.add(Content.from(
                "[简历相关] " + match.embedded().text()
            ));
        }

        return contents;
    }

    /**
     * 根据岗位获取对应的 Namespace
     */
    private String getNamespaceByPosition(String position) {
        switch (position) {
            case "java-backend":
                return "official-java-backend";
            case "frontend":
                return "official-frontend";
            case "fullstack":
            default:
                return "official-java-backend"; // 默认使用 Java 后端
        }
    }

    /**
     * 构建元数据过滤器
     */
    private Map<String, Object> buildMetadataFilter(String position, String difficulty) {
        Map<String, Object> filter = new HashMap<>();
        filter.put("difficulty", difficulty);
        filter.put("type", "official");
        return filter;
    }

    private EmbeddingStore<TextSegment> getStoreByNamespace(String namespace) {
        return PineconeEmbeddingStore.builder()
                .apiKey(System.getenv("PINECONE_API_KEY"))
                .index("interview-assistant-index")
                .nameSpace(namespace)
                .build();
    }

    // 实现 ContentRetriever 接口方法
    @Override
    public List<Content> retrieve(Query query) {
        return retrieveForInterview(query.text(), "java-backend", "medium", null);
    }
}
```

### 为什么选择单 Index 方案？

1. **成本优势**: 只需$30/月，节省 83% 成本
2. **管理便利**: 统一配置，易于维护
3. **检索灵活**: 支持 Namespace + 元数据组合过滤
4. **扩展性强**: 新增题库无需创建新 Index
5. **用户隔离**: 通过 Namespace 实现用户数据隔离
